## Domain Model

All types live in `src/types/index.tsx`. The core entity is `Task`.

**GTD rules enforced in code — do not break these:**
- The main page (`/`) renders `status: "next_action"` tasks only. Inbox, someday, and reference items never appear there.
- Tasks with `isProject: true` are not shown in task lists — only their child actions (tasks with a matching `projectId`) are shown.
- **Important** = a task (or its parent project) has `dueDate` or `scheduledDate` ≤ today. These are surfaced as a top group regardless of area.
- Dates are always `YYYY-MM-DD` ISO strings (`scheduledDate`, `dueDate`). Timestamps are full ISO 8601.

## Translations — Critical Rule

**Every user-facing string must have a key in both `en.json` AND `uk.json`.** Never hardcode UI strings in components.

```typescript
const { t } = useTranslation()
t('status.next_action')                          // static key
t(`area.${area}`, { defaultValue: area })        // dynamic key with fallback
```

Key naming convention: `<domain>.<value>` — e.g. `status.inbox`, `context.deep_work`, `area.work`, `duration.lt15`. See `src/i18n/locales/en.json` for the full key list.

## Styling Rules

| Tool | Use for |
|---|---|
| **Tailwind CSS classes** | Layout, spacing, flexbox — `flex`, `gap-*`, `p-*`, `rounded-*`, etc. |
| **Mantine CSS variables** | Any colour that must adapt to dark mode — `var(--mantine-color-text)`, `var(--mantine-color-body)`, `var(--mantine-color-gray-*)` |
| **Inline styles** | IMPORTANT. Don't use it. Only when computed or conditional values Tailwind cannot express |

- `var(--mantine-color-orange-6)` is reserved for **today / important** highlights — do not use it for other purposes.
- For dark mode branching in component logic, use `useTheme()` from `src/theme.tsx`. Never use Mantine's built-in colour scheme detection; `MantineProvider` always receives `forceColorScheme`.
- Use `cn()` from `src/utils/cn.ts` (clsx + tailwind-merge) for conditional class names.

## Adding New Features — Checklist

1. **GTD alignment** — does this help the user capture, clarify, organise, reflect, or engage?
2. **Types** — add new fields to `src/types/index.tsx`
3. **Translations** — add keys to **both** `en.json` and `uk.json`
4. **Dark mode** — use `useTheme()` or Mantine CSS variables for any new colours
5. **Styling** — Tailwind for layout, Mantine vars for colours
6. **Mock data** — add realistic GTD examples to `src/mockData.ts` that exercise the new feature
7. **Components** — extract to `src/components/` if used in more than one place
